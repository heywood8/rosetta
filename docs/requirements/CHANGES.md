# Requirements Change Log

- 2026-02-15: Initial draft of Rosetta CLI requirements, `configure` command (FR-0001 through FR-0016, NFR-0001 through NFR-0005).
- 2026-02-15: Applied decisions: MCP server name "Rosetta", native CLI tools for claudecode/codex/opencode, windsurf kept as regular draft target, bootstrap bundled from single source of truth.
- 2026-02-15: Added workspace and rosetta.json requirements (FR-0017 through FR-0022): workspace root = CWD, rosetta.json for VCS-safe persistence, no secrets in file, re-use on subsequent runs.
- 2026-02-15: Normalized Windsurf handling in requirements: treated as a regular target, removed duplicate deferred notes, kept single draft-status note in FR-0010.
- 2026-02-16: Renamed "profile" to "MCP preset" (stdio, http). Added "application profile" concept (FR-0023). Extracted FR-0004–FR-0007 to mcp-presets.md, FR-0023 to application-profiles.md. Refactored FR-0010, FR-0012 to reference application profiles. Updated glossary.
- 2026-04-22: rosettify — envelope is now internal. Updated FR-ARCH-0011 to state that the EnrichedEnvelope is never directly serialized for consumers. Added FR-ARCH-0014 (frontend output transformation): frontends extract result on success, sanitized error+help on failure — the envelope wrapper is never the output. Updated FR-CLI-0004 and FR-MCP-0003 accordingly. Added FR-SHRD-0007 (frontend failure logging): all failures logged via shared logger (ERROR for internal_error, WARN for all others) before output is written.
- 2026-05-17: rosettify — added FR-PKG-0005 (Semantic Versioning). package.json `version` SHALL follow semver 2.0.0; MAJOR for any breaking change to CLI/MCP/plan-schema/envelope/error-code contracts, MINOR for backwards-compatible additions (new subcommand, new optional field, new template), PATCH for non-contract bug fixes. Every behavior-changing release SHALL be accompanied by a release-notes entry. Motivation: prevent contract loopholes like the prior FR-PLAN-0024 rename→mutex rewrite from shipping as a silent patch.
- 2026-05-15: rosettify — FR-PLAN-0024 rewritten to mandate a cross-process mutex (atomic-create primitive: mkdir of a lock directory, or O_CREAT|O_EXCL file marker) as the cycle's exclusion gate, and to explicitly forbid plain rename or plain hardlink as the exclusion primitive. Renamed "rename-as-guard" wording to "atomic mutex" throughout. Added a "Concurrency primitive constraints (lessons learned)" block documenting why rename and hardlink are insufficient (verified by multi-process tests: rename lost ~10% of writes at N=10; hardlink lost ~10% at N=30). Acceptance criteria expanded to require an N≥10 concurrent-process test as evidence. The original rename-only formulation was a contract loophole that the first two implementations followed and the unit-test suite could not catch.
- 2026-05-15: rosettify — clarified FR-SHRD-0006 and FR-PLAN-0025: plan-file writes are governed exclusively by FR-PLAN-0024 (atomic-mutex cycle); FR-SHRD-0006 remains the general optimistic-concurrency utility for non-plan JSON files. Aligns intended supersession with explicit requirement text and resolves the prior ambiguity that left both mechanisms claimed for plan writes.
- 2026-05-15: rosettify — plan command extended with templates, atomic write with rename-as-guard, backup chain, and dense JSON output. Modified FR-ARCH-0007 (all output is JSON, no exemptions), FR-HELP-0002 (flat `schemas` dict + `notes` per command), FR-PLAN-0010 / FR-PLAN-0015 (result conforms to compressed-tree shape), FR-PLAN-0016 (restructured help with notes, dual-form examples, schemas dict, templates catalog), FR-PLAN-0017 (added `previous_version` field), FR-PLAN-0018 (dual-form examples mandate), FR-PLAN-0021 (new error codes: invalid_template, missing_template_param, unexpected_template_param, backup_create_failed, plan_file_corrupted), FR-PLAN-0023 (subcommand list updated). Added FR-PLAN-0024 (rename-as-guard atomic write, retention 5, 50-retry cycle restarting from step 1), FR-PLAN-0026 (plan file pretty-formatted on disk), FR-PLAN-0030 (create-with-template), FR-PLAN-0031 (upsert-with-template), FR-PLAN-0032 (list-templates), FR-PLAN-0033 (template registry: two compiled-in TS dicts), FR-PLAN-0034 (placeholder substitution with strict bidirectional matching; canonical kebab-case names span CLI/MCP/template), FR-PLAN-0035 (seed `for-orchestrator` create template), FR-PLAN-0036 (seed `for-subagent` upsert template), FR-PLAN-0040 (compressed-tree output shape for write subcommands), FR-SHRD-0008 (dense JSON output payloads, plan file on disk excepted), FR-SHRD-0009 (read resilience: 100 ms × 50 retries).
